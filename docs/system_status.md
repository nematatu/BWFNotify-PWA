## 🛠️ 最新の主要適用機能・修正内容

### 1. 🗄️ 大規模向け資源効率化と課金防止設計 (完了)
*   **KV読み取りの極小化**: 購読情報（keys, endpoint, exclusions）を KV の **メタデータ** に格納。通知送信時の個別 `kv.get` 読み取りを完全に撤廃（0回）し、`kv.list` 一発での取得に改善（**読取数を 1/1000 以下に圧縮**）。
*   **インテリジェントポーリング**: 画面上にライブ試合が存在しない場合、15秒ごとのライブ得点要求を停止。2分に1回の全体同期のみで待機します。
*   **不活性（Idle）検知**: ブラウザタブが5分以上無操作で放置された場合、自動的にすべてのAPI要求をスリープし、操作再開時に復帰します。

### 2. ⚡ TypeScript 7.0.2 移行 (完了)
*   コンパイラを Go 言語製の高速コンパイラを内蔵した **TypeScript 7.0.2 (最新版)** へ移行。
*   型チェック（`bun run typecheck`）の所要時間を約0.55秒から **約0.11秒 (約5倍高速化)** へ改善。

### 3. 🛡️ 開発フロー自動化 (Husky導入完了)
*   `husky` を導入し、コミット時のプレコミットフック（`.husky/pre-commit`）として `bun run check` を紐付け。
*   コミット前に、型定義整合性・コンパイルチェック・Biomeによるフォーマット/Lint・ユニットテスト/結合テストが自動的に全件パスすることを強制し、バグの混入を未然に防止します。

### 4. 🎨 UI/UX および プッシュ通知の改善 (完了)
*   **通知の選手写真対応**: ペイロードの `icon` にも選手写真URLを設定。`sw.js` 側で `payload.image` が非対応のモバイル端末（iOS等）でもアイコン枠に選手写真がフォールバック表示されるよう改善。
*   **OGP画像の解像度修正**: 512px正方形になっていた `og-image.png` を、正式な 1200x630 の解像度に再レンダリングして置き換え。
*   **UIレイアウトのメリハリ**: 巨大すぎた大会ヘッダー部分をコンパクト化（高さを半分以下に縮小）。「ライブ中バッジ」と「配信を見る（YouTubeリンク）ボタン」のコントラストを大幅に強化して最優先の情報を際立たせました。
