# コントリビューションガイド

BWFNotify PWAへの改善提案とPull Requestを歓迎します。小さく検証可能な変更を優先してください。

## Issueを作成する前に

- 既存Issueに同じ報告や提案がないか確認してください。
- 脆弱性は公開Issueへ投稿せず、[SECURITY.md](SECURITY.md)に従ってください。
- 利用方法の質問だけの場合も、再現環境と確認済みの手順を記載してください。

## 開発手順

1. リポジトリをフォークし、`main` から作業ブランチを作成します。
2. `bun install --frozen-lockfile` で依存関係を導入します。
3. 変更に対応するテストを追加または更新します。
4. `bun run precommit` を実行します。
5. 変更理由、確認方法、画面変更の画像をPull Requestへ記載します。

## 実装上の注意

- 既存のディレクトリ境界と命名規則に合わせてください。
- 無関係なリファクタリングを同じPull Requestへ混ぜないでください。
- ブラウザへ渡す外部URLは検証し、可能な場合は同一オリジンの中継を使用してください。
- KVの読み書き、BWF、YouTubeへの取得回数を増やす変更には、負荷とキャッシュ方針を記載してください。
- Web Pushの購読情報、VAPID秘密鍵、APIキー、実利用者のエンドポイントをコミットしないでください。
- UI変更は320px、390px、1280px、1440pxで表示崩れを確認してください。
- アニメーションには `prefers-reduced-motion` を考慮してください。

## 設定データの変更

日本人選手名や公式YouTube配信元を追加する場合は、公式プロフィール、協会、大会、公式チャンネルなど確認可能な根拠をPull Requestへ記載してください。推測した表記や非公式配信元は追加しないでください。

## テスト

```sh
bun run typecheck
bun run lint
bun run test:unit
bun run test:integration
bun run test:layout
bun run dry-run
```

画面変更ではPlaywrightテストを追加し、得点、選手名、画像欠損、最小幅など動的状態も確認してください。

## Pull Request

- 1つのPull Requestは1つの目的に限定してください。
- 破壊的変更や運用設定の変更はタイトルと本文で明示してください。
- テストできなかった項目がある場合は理由を記載してください。
- レビュー指摘への対応後は、必要な検査を再実行してください。

Pull Requestを送信すると、提出したコードを本リポジトリのMIT Licenseで公開する権利があることに同意したものと扱います。
